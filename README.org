#+title: komga-reader
#+author: 

Browse and read books from a [[https://komga.org][Komga]] server inside Emacs.

* Features

- 📚 *Library browser* — Tabulated list of all books with title, author, page count, read progress, and last-read timestamp.
- 📖 *Table of contents* — Quick chapter navigation for each book.
- 📄 *HTML reader* — Renders book chapters with =shr= (images optional via =komga-reader-render-images=).
- 📜 *Reading history* — Navigate back and forth through visited chapters with ~l~/~r~, like =Info-history-back/forward=.
- 🔄 *Progress sync* — Automatically saves and resumes your reading position via Komga's progression API (synced on quit via =quit-window-hook=).
- ⚡ *Preloading* — Upcoming chapters are fetched in the background for instant page turns.
- 💾 *Cross-session caching* — Book list is cached via =multisession= for instant startup (Emacs 29+).
- ▶️ *Resume reading* — Jump back to your last book with ~M-x komga-reader-resume~.
- 🔧 *Fully configurable* — Server URL, API key, device ID/name, cache TTL, debug logging, and curl extra args are all =defcustom= variables (with optional environment-variable fallback).
- 🐛 *Debug logging* — Toggle =komga-reader-debug= to trace HTTP requests, responses, and reader lifecycle in the =*Messages*= buffer.
- 🔗 *Curl extra args* — Pass arbitrary flags (e.g. proxy =-x=) via =komga-reader-curl-extra-args= for every HTTP request.

* Installation

** Requirements

- Emacs 29.1+ (uses =json-serialize= / =json-parse-string=)
- A running [[https://komga.org][Komga]] server
- =curl= (used for HTTP requests)

** Manual

#+begin_src bash
git clone https://github.com/yourusername/komga-reader.git
# Add to load-path, then:
#+end_src

#+begin_src elisp
(add-to-list 'load-path "/path/to/komga-reader")
(require 'komga-reader)
#+end_src

** with ~use-package~

#+begin_src elisp
(use-package komga-reader
  :load-path "~/path/to/komga-reader"
  :custom
  (komga-reader-komga-url "http://localhost:25600")
  (komga-reader-komga-api-key "YOUR_API_KEY")
  (komga-reader-komga-device-id "emacs-desktop")
  (komga-reader-komga-device-name "Emacs Desktop")
  (komga-reader-booklist-cache-ttl 300)
  (komga-reader-debug nil)
  (komga-reader-curl-extra-args nil)
  :commands (komga-reader komga-reader-resume))
#+end_src

* Configuration

Two custom variables are required to connect to your Komga server:

| Variable                      | Type    | Description                          | Fallback            |
|-------------------------------+---------+--------------------------------------+---------------------|
| ~komga-reader-komga-url~      | string  | Base URL of the Komga server         | ~KOMGA_URL~ env var |
| ~komga-reader-komga-api-key~  | string  | API key for authentication           | ~KOMGA_API_KEY~ env |

Set them in your init file:

#+begin_src elisp
(setq komga-reader-komga-url "https://komga.example.com")
(setq komga-reader-komga-api-key "xxx-yyy-zzz")
#+end_src

Or interactively via ~M-x customize-group RET komga-reader RET~.

Optional settings:

| Variable                              | Default                  | Description                           |
|---------------------------------------+--------------------------+---------------------------------------|
| ~komga-reader-preload-chapters-count~ | 3                        | Number of upcoming chapters to cache  |
| ~komga-reader-komga-device-id~        | =emacs-(system-name)=    | Device ID for progress sync           |
| ~komga-reader-komga-device-name~      | =Emacs (system-name)=    | Device name for progress sync         |
| ~komga-reader-booklist-cache-ttl~     | 300                      | Cache TTL in seconds (0 = disable)    |
| ~komga-reader-debug~                  | =nil=                    | Enable debug logging                  |
| ~komga-reader-curl-extra-args~        | =nil=                    | Extra curl flags (e.g. proxy)         |
| ~komga-reader-render-images~          | =nil=                    | Show images in chapter HTML           |

** Debugging and proxying

Enable debug logging to trace every HTTP request and response:

#+begin_src elisp
(setq komga-reader-debug t)
#+end_src

Pass extra arguments to curl (e.g. for proxying through Burp or mitmproxy):

#+begin_src elisp
(setq komga-reader-curl-extra-args '("-x" "http://127.0.0.1:8080"))
;; or ignore SSL certificate errors for self-signed proxies:
(setq komga-reader-curl-extra-args '("--insecure"))
#+end_src

* Usage

** Opening the library

Run ~M-x komga-reader RET~ to open the book list.

When the list is opened, if a cached book list exists (and the cache TTL has not expired), it is shown immediately while fresh data is fetched in the background. If the list is from cache and you open a book, a message warns you that the progress shown may not be up-to-date. Press ~g~ to refresh the list at any time.

Run ~M-x komga-reader-resume RET~ to jump directly back to the last book you were reading (requires Emacs 29+).

** Book list keys (~komga-reader-booklist-mode~)

| Key     | Action                               |
|---------+--------------------------------------|
| ~RET~   | Open selected book at saved progress |
| ~t~     | Open table of contents for book      |
| ~g~     | Refresh book list                    |
| ~q~     | Quit window                          |

** Reader keys (~komga-reader-reader-mode~)

| Key         | Action                           |
|-------------+----------------------------------|
| ~n~         | Next chapter                     |
| ~p~         | Previous chapter                 |
| ~l~         | History back                     |
| ~r~         | History forward                  |
| ~SPC~       | Scroll down                      |
| ~DEL~       | Scroll up                        |
| ~t~         | Open TOC, jump to current chapter|
| ~g~         | Refresh / fetch server progress  |
| ~→~         | Next chapter                     |
| ~←~         | Previous chapter                 |
| ~q~         | Quit and sync progress           |

** Table of contents keys (~komga-reader-toc-mode~)

| Key   | Action              |
|-------+---------------------|
| ~RET~ | Open chapter        |
| ~q~   | Quit window         |

* Architecture

#+begin_example
komga-reader.el          — UI: book list, TOC, entry point
komga-reader-backend.el  — Async curl abstraction + backend dispatch
komga-reader-komga.el    — Komga REST API implementation
komga-reader-reader.el   — Chapter rendering + reader mode
#+end_example

Backends register themselves by setting ~komga-reader-backend-impl~. Currently only Komga is implemented, but the abstraction allows other servers that expose a compatible API.
